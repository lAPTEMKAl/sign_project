# План итогового отчёта (соответствует методическим рекомендациям, раздел 8)

См. также готовый шаблон в формате Word: `Отчёт_по_практике_ШАБЛОН.docx`
(оформлен по ГОСТ в соответствии с "Методические_рекомендации_CV_ГОСТ").

1. Титульный лист (по ГОСТ-шаблону вуза)
2. Введение / постановка задачи
   - Что автоматизируется: распознавание класса дорожного знака по изображению
   - Почему это полезно: применимо в ADAS, видеоаналитике дорожной обстановки, автонавигации
3. Описание данных
   - Источник: RTSD (Russian Traffic Sign Dataset)
   - Объём выборки, количество классов, тип разметки (классификация по кропам знаков)
   - Ограничения использования данных, лицензия
   - Примеры изображений (в т.ч. мелкие/размытые/плохо освещённые — отдельным блоком)
4. Предобработка данных
   - train/val/test split (доли, seed, файл split_seed.txt)
   - Проверка разметки и баланса классов (class_stats.csv)
   - Аугментации (см. src/datasets.py): блюр, джиттер яркости/контраста, перспектива,
     RandomErasing — имитация сложных условий съёмки
5. Выбранные архитектуры и обоснование
   - Таблица 5 архитектур (ResNet18, MobileNetV3-Small, EfficientNet-B0, DenseNet121, ViT-B/16)
   - Почему каждая подходит/не подходит для задачи (см. README.md)
6. Экспериментальное сравнение
   - Таблица сравнения (runs/experiment_1/comparison_table.md) — input size, эпохи,
     test accuracy, F1, время инференса, размер модели
   - Confusion matrix лучшей модели (runs/experiment_1/eval_best/confusion_matrix.png)
   - Топ путаемых пар похожих знаков (top_confused_pairs.json)
7. Выбор лучшей модели
   - Обоснование не только по accuracy, но и по скорости/размеру для целевого сценария
     (например, если планируется real-time видеоаналитика — приоритет MobileNetV3/EfficientNet
     даже при небольшом отставании по accuracy от ViT)
8. Примеры работы модели
   - Минимум 3 успешных примера (runs/experiment_1/eval_best/examples_correct)
   - Минимум 3 ошибочных примера с разбором причин (examples_wrong)
9. Демонстрационный модуль
   - Скриншоты Streamlit-приложения (demo/app.py): загрузка фото, класс, уверенность,
     топ-3, статистика истории запросов
   - Описание хранения истории (SQLite, runs/history.db) и генерации отчёта (PDF/Excel)
10. Выводы
    - Можно ли использовать систему в реальных условиях
    - Что нужно улучшить (например: расширить датасет редкими классами,
      добавить TTA для повышения устойчивости к размытию)
11. Список использованных источников
12. Приложения (полные таблицы метрик, исходный код — ссылка на репозиторий)
